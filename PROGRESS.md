**Урок 3 — 2026-06-22 — Цикл for, список list, range()**

Файл: `month1/week1/lesson3.py`

Что разобрали:
- **Список (list)** — коллекция значений в одной переменной, пишется в `[]`: `cities = ["Москва", "Казань"]`
- **`for` цикл** — проходит по каждому элементу коллекции: `for city in cities: print(city)`
- Именование: коллекция — во множественном числе (`cities`), элемент — в единственном (`city`)

Что понял сам:
- `for x in список` — на каждом шаге `x` это один элемент
- `range(1, 11)` даёт 1..10, не 1..11 — последнее число не включается

Ошибки которые допустил:
- Написал `=` вместо `==` при сравнении в `if` — снова та же ошибка что в уроке 2

## МОИ ОШИБКИ И ПАТТЕРНЫ

> Сюда записываются повторяющиеся ошибки, неверные понимания, ловушки в которые попадаю

**2026-06-18 — Урок 1: типы данных**

- Ошибка: написал `age = "27"` в кавычках, поэтому возраст стал строкой `str`, а не числом `int`.
- Причина: в Python кавычки превращают значение в текст, даже если внутри записаны цифры.
- Правильный вариант: `age = 27`, если значение должно участвовать в числовых операциях.
- Паттерн для проверки: если значение планируется считать, сравнивать как число или использовать в формулах — не ставить кавычки и проверить через `type()`.

**2026-06-22 — Уроки 2–3: сравнение в условиях**

- Ошибка: несколько раз написал `=` вместо `==` при проверке условия.
- Причина: `=` присваивает значение, а `==` сравнивает два значения.
- Правильный вариант: в `if`, `elif` и фильтрации внутри циклов использовать `==`, если нужно проверить равенство.
- Паттерн для проверки: если строка начинается с `if` или `elif`, внутри условия почти всегда должно быть сравнение (`==`, `!=`, `>`, `<`, `>=`, `<=`), а не присваивание.

**2026-06-22 — Урок 4: бесконечный цикл**

- Ошибка: забыл менять переменную внутри `while`, из-за чего цикл стал бесконечным.
- Причина: `while` сам не двигает счётчик; условие должно когда-нибудь стать `False`.
- Правильный вариант: внутри цикла явно менять переменную, например `number += 1`.
- Паттерн для проверки: в каждом `while` найти переменную из условия и убедиться, что она изменяется внутри тела цикла.
