# Дорожная карта обучения AI/ML

## Финальная цель

За 7–10 лет выйти на уровень `AI Industrial Architect / Research Engineer`.

Специалист такого уровня должен уметь:

- проектировать сложные AI-системы;
- работать с данными, моделями и программной инфраструктурой;
- внедрять AI в реальные физические и производственные процессы;
- понимать ограничения оборудования, задержек и надёжности;
- проводить и воспроизводить эксперименты;
- принимать архитектурные решения и отвечать за систему целиком.

## Режим обучения

Рабочий график: `2/2`.

Рабочие дни:

- до 30 минут;
- краткая теория;
- одна небольшая практическая задача;
- повторение терминов и чтение кода.

Выходные дни:

- до 6 часов;
- изучение новой темы;
- решение задач;
- инженерная практика;
- проектная работа;
- повторение и проверка понимания.

Основная среда обучения — локальный компьютер и сервер. Облачные
сервисы подключаются только тогда, когда они действительно нужны.

- отладка, тестирование и документирование;
- архитектура и качество кода.

### Инженерные инструменты

- Git;

- командная строка;
- Linux и WSL;
- виртуальные окружения;
- SQL и базы данных;
- Docker;
- API;
- основы CI/CD;
- логирование и мониторинг.

### Данные и математика

- NumPy;
- pandas;
- визуализация;
- очистка и проверка данных;
- описательная статистика;
- теория вероятностей;
- линейная алгебра;
- основы математического анализа;

### Machine Learning

- постановка ML-задачи;
- подготовка данных;
- baseline;
- регрессия;
- классификация;
- деревья решений и ансамбли;

- снижение размерности;

- воспроизводимость экспериментов;
- введение в нейронные сети.

### Результат Master Core

Самостоятельный `ML/MLOps Engineer`, который может:

- написать и протестировать программу;
- обработать реальные данные;
- обучить и корректно оценить модель;
- создать воспроизводимый ML-пайплайн;
- упаковать и запустить систему;
- объяснить принятые технические решения.

## Этап 2. xAI Core

Ориентировочная продолжительность: 3–5 годы.

Этот этап включается поверх готового Master Core, а не вместо него.

### Основные направления

- архитектура трансформеров;
- Large Language Models;
- embeddings;
- vector databases;
- Retrieval-Augmented Generation;
- evaluation LLM-систем;
- prompt engineering как инженерная дисциплина;
- tool calling;
- agentic workflows;
- multi-agent systems;
- безопасность и ограничения AI-систем;
- Explainable AI;
- multimodal AI;
- оптимизация и локальный запуск моделей;
- MLOps для генеративных моделей.

### Исследовательская подготовка

- чтение научных статей;
- разбор методологии эксперимента;
- воспроизведение результатов;
- абляционные исследования;
- сравнение с baseline;
- анализ ошибок;
- технические отчёты;
- формулирование и проверка гипотез.

### Результат xAI Core

`AI Engineer / Research-oriented Engineer`, который способен:

- проектировать сложные интеллектуальные системы;
- оценивать их качество и ограничения;
- выбирать архитектуру на основании экспериментов;
- разбираться в научных публикациях;
- создавать локальные LLM-, RAG- и agentic-системы.

## Git, GitHub и Kaggle

Git изучается в первый месяц параллельно с Python.

GitHub подключается после появления первых осмысленных решений и
мини-проектов. Репозитории должны показывать развитие инженерных навыков,
а не быть хранилищем случайных упражнений.

Kaggle подключается после освоения основ Python, NumPy и pandas:

1. изучение готовых notebooks и датасетов;
2. самостоятельный анализ данных;
3. учебные соревнования;
4. полноценные соревнования и командная работа.

Kaggle является инструментом практики, но не заменяет разработку
полноценных программных и ML-систем.

## Правила перехода между этапами

Переход выполняется по фактическим навыкам, а не по прошедшему времени.

К xAI Core можно переходить, когда:

- Python используется уверенно;
- есть опыт работы с данными;
- освоены Git и основы инженерной разработки;
- понятны базовые ML-алгоритмы и метрики;
- завершены воспроизводимые проекты.

К глубокой Industrial Core специализации можно переходить, когда:

- есть опыт построения законченных AI-систем;
- понятны ограничения моделей и инфраструктуры;
- сформировано системное инженерное мышление;
- можно самостоятельно ставить и проверять технические гипотезы.

## Текущая точка

- направление: `Master Core`;
- уровень программирования: Python с нуля;
- текущий урок: переменные, типы и операции;
- следующий управляющий сигнал: `НАЧИНАЕМ`.

## Команды учебного процесса

- `НАЧИНАЕМ` — начать следующий урок;
- `ГОТОВО` — провести проверку и код-ревью;
- `СТОП` — зафиксировать текущую точку;
- `ПРОДОЛЖИМ` — продолжить с последнего зафиксированного места.
