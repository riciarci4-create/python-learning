# Урок 5 — Git: `init`, `add`, `commit`

Дата: 2026-06-22
Формат: терминальная практика, отдельного Python-файла нет.
Статус: базовый цикл закреплён; состояния Git повторно проверены 2026-07-20.

## Цель

Создать локальный репозиторий, осознанно выбрать изменения и сохранить локальный
снимок с понятным описанием.

## Теория

Git хранит историю состояний проекта. В базовом цикле участвуют три области:

1. **Working tree** — текущие файлы и незакоммиченные изменения.
2. **Staging area** — изменения, выбранные для следующего коммита.
3. **Commit** — сохранённый локальный снимок подготовленных изменений.

`git add` не создаёт коммит и не отправляет код в интернет. `git commit` не
отправляет изменения на GitHub: он фиксирует их в локальной истории.

## Основные команды

```powershell
git init
git status
git add path/to/file.py
git commit -m "add machine status validation"
git log
```

`git add .` выбирает все видимые изменения и поэтому требует предварительной
проверки `git status`. Для учебной инженерной практики предпочтительно явно
добавлять нужные пути.

## Микросценарии

- Изменили два файла, добавили один — в коммит попадёт только staged-файл.
- После `git add` файл ещё можно изменить: staged и рабочая версии могут различаться.
- После успешного коммита staging area очищается, но история сохраняется.

## Инженерный смысл

Коммит является проверяемым checkpoint: он должен представлять одно осмысленное
изменение. История помогает искать причину регрессии, проводить review и безопасно
возвращаться к предыдущему состоянию.

## Что было понято и исправлено

- Аналогия: `add` выбирает кадр, `commit` делает снимок.
- `git add.` без пробела не распознаётся как нужная команда.
- Перед первым коммитом потребовалась настройка имени и email.
- На месячном контроле различие working tree / staging / commit потребовало
  повторения и затем было воспроизведено на новом примере.

## Math Core

Состояния можно представить переходами:
`working tree -> staging area -> commit`. Коммит не стирает прошлое, а добавляет
новую вершину истории.

## English Core

- **repository** — репозиторий;
- **working tree** — рабочее дерево;
- **staging area** — область подготовки;
- **commit** — зафиксированный снимок;
- **history** — история;
- **status** — состояние.

## Anki

1. **Каков базовый цикл Git?** — Изменить → `git status` → `git add <пути>` → проверить → `git commit`.
2. **Что делает `git status`?** — Показывает ветку и состояния изменённых, staged и новых файлов.
3. **Чем `add` отличается от `commit`?** — `add` выбирает изменения, `commit` фиксирует выбранный снимок локально.
4. **Как посмотреть историю?** — `git log`.

## Урок усвоен, если

- различаешь working tree, staging area и commit;
- добавляешь только нужные файлы;
- пишешь осмысленное сообщение коммита;
- не путаешь локальный commit с публикацией на GitHub.
