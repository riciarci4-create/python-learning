## Цель

Перехватывать ожидаемые сбои на границах программы и сохранять понятное,
контролируемое поведение.

## Теория

`try` выполняет код. Он не проверяет его заранее. Если во время выполнения
возникает исключение подходящего типа, управление переходит в `except`.

```python
try:
    count = int(raw_count)
except ValueError:
    count = None
```

Один `try` может иметь несколько обработчиков. Следует ловить конкретные типы,
потому что широкий `except Exception` скрывает причину и усложняет диагностику.

## Важные исключения

- `ValueError` — тип допустим, но значение нельзя преобразовать или использовать;
- `TypeError` — операция получила несовместимый тип;
- `ZeroDivisionError` — попытка деления на ноль;
- `FileNotFoundError` — нужный файл отсутствует.

## Микропримеры

```python
int("sensor_error")          # ValueError
"error" + 5                 # TypeError
10 / 0                      # ZeroDivisionError
```

После ошибки присваивание может не завершиться. Поэтому нельзя без проверки
возвращать переменную, которая создавалась только внутри неуспешного `try`.

## Инженерный смысл

Исключения обрабатывают ожидаемые сбои ввода-вывода: неверный ввод, отсутствующий
файл, недоступный ресурс. Они не должны маскировать ошибки программиста и не
заменяют обычные условия для предсказуемых состояний.

## Что было понято и исправлено

- Без обработки ожидаемая ошибка аварийно завершает сценарий.
- Python проверяет обработчики сверху вниз.
- Ввод и результат деления сначала были смешаны в одной строке; переменные разделены.
- Уточнено: `ValueError` — не «ошибка типа», а неподходящее значение допустимого типа.

## Math Core

Деление `a / b` имеет область определения `b != 0`. Обработка исключения является
альтернативной ветвью вычисления для входа вне допустимой области.

## English Core

- **exception** — исключение;
- **handle** — обработать;
- **raise** — выбросить исключение;
- **catch** — перехватить;
- **crash** — аварийно завершиться;
- **traceback** — цепочка вызовов ошибки.

## Anki

1. **Что произойдёт без `try/except` при `int("abc")`?** — Возникнет необработанный `ValueError`, сценарий остановится.
2. **Чем `ValueError` отличается от `TypeError`?** — Первое про недопустимое значение, второе про несовместимый тип.
3. **Можно ли иметь несколько `except`?** — Да, для разных ожидаемых исключений.
4. **Почему опасен `except Exception`?** — Он может скрыть непредусмотренную ошибку и её причину.
