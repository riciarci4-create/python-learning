# Урок 9 — GitHub: remote и push

Дата: 2026-06-23
Формат: терминальная практика, отдельного Python-файла нет.
Статус: пройден; публикация веток повторно применена в Pull Request.

## Цель

Связать локальный Git-репозиторий с GitHub и опубликовать локальные коммиты.

## Теория

Git работает локально. GitHub хранит удалённую копию репозитория и предоставляет
Pull Request, review и публичное портфолио.

```powershell
git remote add origin <repository-url>
git branch -M main
git push -u origin main
```

`origin` — обычное имя удалённого репозитория. `-u` настраивает upstream-связь,
после чего часто достаточно `git push`.

## Полный базовый цикл

```powershell
git status
git add <paths>
git commit -m "describe completed change"
git push
```

`push` отправляет уже существующие локальные коммиты. Незакоммиченные изменения
не публикуются.

## Инженерный смысл

GitHub-профиль показывает не только код, но и историю решений, README и качество
коммитов. Нельзя публиковать API-ключи, `.env`, пароли, виртуальные окружения и
случайные данные.

## Что было понято и исправлено

- Первый push требует указания remote и ветки.
- Последующие push используют настроенный upstream.
- README является публичным входом в проект.
- GitHub не заменяет локальный Git: commit создаётся локально до push.

## Math Core

Локальная и удалённая истории можно рассматривать как два состояния графа
коммитов. Push переносит отсутствующие удалённо локальные вершины.

## English Core

- **remote repository** — удалённый репозиторий;
- **origin** — принятое имя основного remote;
- **push** — отправить коммиты;
- **upstream** — отслеживаемая удалённая ветка;
- **main branch** — основная ветка;
- **README** — публичное описание проекта.

## Anki

1. **Что отправляет `git push`?** — Локальные коммиты, которых ещё нет в remote.
2. **Что делает `git remote add origin`?** — Сохраняет адрес удалённого репозитория под именем `origin`.
3. **Зачем `-u` при первом push?** — Настроить отслеживание удалённой ветки.
4. **Попадут ли незакоммиченные изменения на GitHub?** — Нет.

## Урок усвоен, если

- различаешь commit и push;
- проверяешь remote и текущую ветку;
- публикуешь только проверенные коммиты;
- понимаешь роль README и `.gitignore`.
