## Цель

Проверять программу слоями, исправлять причины ошибок в порядке риска и
превращать работающий скрипт в более предсказуемую инженерную систему.

## Порядок review

1. **Correctness** — выдаёт ли код правильный результат.
2. **Robustness** — обрабатывает ли ожидаемые неверные входы и сбои.
3. **Security / reproducibility** — нет ли секретов и можно ли повторить запуск.
4. **Maintainability / readability** — понятны ли ответственности, имена и поток данных.
5. Стиль — последним, если он не скрывает дефект.

## Серьёзность замечаний

- **blocker** — нельзя объединять или выпускать;
- **major** — существенный дефект поведения или архитектуры;
- **minor** — локальное улучшение качества;
- **nit** — необязательная мелочь стиля.

## Return contract

Функция должна иметь понятный набор возможных результатов. `divide()` возвращает
число или `None`; `analyze_log()` — словарь счётчиков или `None`. Вызывающий код
проверяет `None` и формирует пользовательское сообщение.

Смешение числа и строки ошибки приводит к ошибкам вроде `"error" + 5` и усложняет
все места вызова.

## Побочные эффекты и точка входа

Печать, запись файла и изменение внешнего состояния — side effects. Код верхнего
уровня выполняется при импорте. Защита:

```python
if __name__ == "__main__":
    main()
```

позволяет запускать основной сценарий напрямую, но не выполнять его автоматически
при импорте модуля.

## Разделение ответственностей

- `analyze_log(log_path)` читает и считает, затем возвращает данные;
- `main()` строит пути, организует процесс, формирует и записывает отчёт.

Результат функции сохраняется в переменную и не вычисляется повторно, особенно
если вызов читает файл или создаёт другой побочный эффект.

## Math Core

- Input partitioning: корректные, граничные, неверного типа и неподдерживаемые входы.
- Инвариант счётчиков: `0 <= event_count <= line_count`.
- Проверка классов входов заменяет невозможный полный перебор всех значений.

## English Core

- **correctness** — корректность результата;
- **robustness** — устойчивость;
- **side effect** — побочный эффект;
- **return contract** — контракт возвращаемых значений;
- **main guard** — защита точки входа;
- **entry point** — точка входа;
- **severity** — серьёзность замечания;
- **invariant** — сохраняющееся условие.

## Anki

1. **В каком порядке проводить review?** — Корректность, устойчивость и безопасность; затем архитектура и читаемость; стиль последним.
2. **Зачем main guard?** — Не запускать основной сценарий автоматически при импорте.
3. **Зачем понятный return contract?** — Чтобы вызывающий код знал возможные типы и обрабатывал каждый исход.
4. **Как проверить множество входов?** — Разделить на классы, проверить границы и контролировать инварианты.
