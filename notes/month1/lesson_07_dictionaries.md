## Цель

Хранить именованные значения и обращаться к ним по смысловому ключу, а не по
позиции.

## Теория

Словарь хранит пары `ключ: значение`.

```python
machine = {
    "name": "CNC-1",
    "status": "running",
    "part_count": 120,
}
```

Доступ выполняется по ключу: `machine["status"]`. Присваивание по существующему
ключу изменяет значение, по новому — добавляет пару. `del` удаляет пару.

## Обход словаря

```python
for key in machine:
    print(key)

for value in machine.values():
    print(value)

for key, value in machine.items():
    print(key, value)
```

Имя переменной цикла не влияет на поведение, но должно передавать смысл.

## Когда выбирать структуру

- `list` — изменяемая последовательность;
- `tuple` — фиксированная позиционная запись;
- `dict` — именованные связи `ключ -> значение`.

## Инженерный смысл

Словари подходят для конфигураций, результатов анализа, параметров станка и
JSON-подобных сообщений. Ошибка в имени ключа приводит к `KeyError`, поэтому
ключи являются частью контракта данных.

## Что было понято и исправлено

- Обычный `for` по словарю проходит по ключам.
- Значения и пары получают через `.values()` и `.items()`.
- На месячном экзамене `dict` был перепутан с `list` и `tuple`; после коррекции
  словарь правильно выбран для состояний станков и именованных счётчиков.

## Math Core

Словарь реализует отображение: каждому ключу соответствует значение. Например,
`event_name -> event_count`.

## English Core

- **dictionary** — словарь;
- **key** — ключ;
- **value** — значение;
- **key-value pair** — пара ключ–значение;
- **item** — элемент словаря;
- **mapping** — отображение.

## Anki

1. **Когда выбирать `dict`?** — Когда значения должны иметь смысловые имена-ключи.
2. **Что делает `data["count"]`?** — Возвращает значение по точному ключу `"count"`.
3. **Чем `.values()` отличается от `.items()`?** — Первое даёт значения, второе — пары ключ и значение.
4. **Что происходит при присваивании новому ключу?** — В словарь добавляется новая пара.
