## Цель

Строить ветвление, правильно упорядочивать условия и отличать присваивание от
сравнения.

## Теория

`if` проверяет первое условие. `elif` проверяется только если предыдущие ветви
не сработали. `else` обрабатывает все оставшиеся случаи. В одной цепочке
выполняется только первая ветвь с результатом `True`.

```python
if temperature >= 90:
    print("alarm")
elif temperature >= 70:
    print("normal")
else:
    print("cold")
```

Python обозначает блок двоеточием и отступом. `=` присваивает значение, `==`
сравнивает. `input()` всегда возвращает `str`, даже если пользователь ввёл цифры.

## Микропримеры

```python
is_running = status == "running"
```

```python
if pressure > limit:
    print("Превышение")
```

```python
raw_age = input("Возраст: ")
age = int(raw_age)
```

Более узкое или строгое условие должно стоять раньше общего. Если сначала
проверить `age >= 18`, ветвь `age >= 65` никогда не будет достигнута.

## Инженерный смысл

Условия превращают измерения в решения: норма, предупреждение, авария. Ошибка в
порядке порогов может скрыть аварийную ветвь, поэтому диапазоны должны быть
взаимоисключающими или явно приоритизированными.

## Что было понято и исправлено

- Python идёт сверху вниз и останавливается на первой истинной ветви.
- `elif age >= 65` был расположен после `age >= 18`; порядок исправлен.
- `=` внутри условия был заменён на `==`.

## Math Core

Условия задают интервалы числовой оси. Например, `temperature >= 90` описывает
множество всех температур от 90 и выше. Правильная цепочка должна покрывать
нужную область без недостижимых ветвей.

## English Core

- **condition** — условие;
- **comparison** — сравнение;
- **branch** — ветвь выполнения;
- **true / false** — истина / ложь;
- **assignment** — присваивание;
- **threshold** — порог.

## Anki

2. **Почему важен порядок `if / elif`?** — Выполняется первая истинная ветвь; общее условие может скрыть частное.
3. **Что возвращает `input()`?** — Всегда строку `str`.
4. **Что обязательно после `if x > 5`?** — Двоеточие `:` и блок с отступом.
