## Цель

Объединить связанные данные и операции в объект, различать класс, экземпляр,
атрибут и метод и управлять состоянием «не загружено / загружено».

## Основные понятия

- **Класс** — шаблон объектов.
- **Объект / экземпляр** — конкретный созданный представитель класса.
- **Атрибут** — данные конкретного объекта.
- **Метод** — функция, определённая внутри класса.
- `self` — ссылка на конкретный объект текущего вызова.

```python
class ImageLoader:
    def __init__(self, image_path):
        self.image_path = image_path
        self.image = None
```

Параметр `image_path` существует во время вызова `__init__`. Атрибут
`self.image_path` сохраняется в состоянии объекта для последующих методов.

## Состояние объекта

До `load()` значение `self.image` равно `None`: объект знает путь, но изображение
ещё не открыто. После успешной загрузки атрибут хранит объект Pillow Image.

```text
created -> path known, image is None
loaded  -> path known, image contains opened image
```

Каждый экземпляр хранит своё состояние. Изменение `loader_1.image_path` не меняет
`loader_2.image_path`.

## Ответственности методов

- `exists()` отвечает на вопрос и возвращает `bool`;
- `load()` выполняет загрузку и меняет состояние;
- `get_info()` возвращает данные или `None`, если изображения нет.

Метод-запрос предпочтительно возвращает значение, а не печатает его: вызывающий
код сам решает, как использовать результат.

## Защита от неправильного состояния

Перед обращением к `.size` и распаковкой результата нужно проверить отсутствие
данных. Иначе `None` будет использован как изображение или кортеж.

## Что было понято и исправлено

- `self` сначала ошибочно воспринимался как класс; закреплено, что это конкретный объект.
- `show_path()` был перегружен проверкой существования; ответственности разделены.
- `exists()` исправлен с печати на `return`.
- Код загрузки оказался после раннего `return False`; отступ исправлен.
- `get_info()` при отсутствии изображения возвращает `None`, а не строку другого типа.

## Math Core

Объект можно представить вектором состояния `(image_path, image)`. Методы
выполняют переходы между допустимыми состояниями. `None` является явным маркером
отсутствия данных.

## Anki

1. **Чем `image_path` отличается от `self.image_path`?** — Первое является параметром, второе — сохранённым атрибутом конкретного объекта.
2. **Что хранит `self.image` до и после `load()`?** — Сначала `None`, затем открытое изображение.
3. **Почему `exists()` возвращает `bool`?** — Это метод-запрос «да/нет», результат нужен вызывающему коду.
4. **Зачем проверять `info is None`?** — Чтобы не распаковывать отсутствующие данные.
