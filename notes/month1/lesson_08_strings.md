## Цель

Нормализовать, очищать, разделять и проверять текстовые данные.

## Теория

`str` — неизменяемая последовательность символов. Методы строк возвращают новую
строку и не изменяют исходную.

```python
raw_status = "  Cnc Started  "
clean_status = raw_status.strip().upper()
```

Основные методы:

- `.upper()` — верхний регистр;
- `.lower()` — нижний регистр;
- `.strip()` — убрать пробелы и переносы по краям;
- `.replace(old, new)` — заменить подстроку;
- `.split(separator)` — разделить строку на список частей.

## Микропримеры

```python
len("CNC 1")              # 5, пробел тоже символ
"CNC" in "CNC started"   # True
"a,b,c".split(",")       # ["a", "b", "c"]
"error".upper()           # "ERROR"
```

Нормализация регистра позволяет сравнивать `CNC`, `cnc` и `Cnc` одинаково.

## Инженерный смысл

Текст из логов и файлов часто содержит разные регистры, пробелы и разделители.
Перед сравнением его приводят к согласованной форме. Но чрезмерная очистка может
стереть значимую информацию, поэтому правило нормализации должно быть явным.
