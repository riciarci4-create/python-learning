# Learning System

> Status: archived planning baseline. For the actual lesson order and current
> rules, use local `PROGRESS.md`, `CLAUDE.md`, and `state/learning_state.yaml`.

## Goal

Build a long-range trajectory from zero in programming to an engineer capable of designing, implementing, and eventually architecting AI systems for real industrial environments.

## Strategic Path

### 1. Master Core

Focus:

- Python;
- computational thinking;
- data structures and algorithms;
- Linux, Git, Docker;
- software engineering basics;
- mathematics for ML;
- classical machine learning;
- deployment and engineering discipline.

Outcome:

- independent `ML / MLOps Engineer` foundation.

### 2. xAI Core

Focus:

- LLM systems;
- RAG;
- agentic workflows;
- explainability;
- evaluation and reproducibility;
- applied research habits.

Outcome:

- `AI Engineer` with system-level reasoning, not tool-level imitation.

### 3. Industrial Core

Focus:

- AI for manufacturing;
- digital twins;
- robotics;
- edge AI;
- integration of software, data, and physical processes.

Outcome:

- differentiated path toward `AI Industrial Architect / Research Engineer`.

## Learning Principles

- no topic skipping;
- theory only in the amount required for implementation;
- code first when possible;
- every lesson must have measurable output;
- mistakes are analyzed by cause, not only corrected;
- practical engineering framing is preferred over toy examples.

## Training Rhythm

### Work Days

- micro-theory;
- one compact coding task;
- one short review or terminology block.

Target duration: `<= 30 min`.

### Off Days

- focused lesson work;
- exercise block;
- engineering task;
- review and checkpoint update;
- English terminology expansion.

Target duration: up to `6 h`.

## Gate Logic

Move to the next layer only when the previous one is operational, not merely familiar.

- `Master Core -> xAI Core`: only after solid Python, data work, Git, reproducible projects, and basic ML.
- `xAI Core -> Industrial Core deep specialization`: only after system-building habits and enough AI maturity to evaluate tradeoffs realistically.
